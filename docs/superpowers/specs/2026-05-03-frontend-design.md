# USMP 前端页面设计规格说明书

**日期**: 2026-05-03
**版本**: v1.0
**设计范围**: 完整平台前端页面设计

## 一、整体设计原则

### 1.1 设计风格
- **企业级专业风格**: 采用 Element Plus 蓝灰色系，强调数据可读性和操作便捷性
- **操作优先**: 功能优先，减少不必要的视觉装饰
- **一致性**: 所有模块采用统一的交互模式和视觉语言

### 1.2 适配策略
- 仅桌面端优先（1024px+），移动端后续迭代
- 不做响应式移动端适配

## 二、整体布局设计

### 2.1 页面结构

```
┌─────────────────────────────────────────────────────────────────┐
│  HEADER: Logo | 全局搜索 | 设备状态指示器 | 通知 | 用户 | 主题切换 │
├──────────┬──────────────────────────────────────────┬───────────┤
│          │                                          │           │
│  SIDEBAR │              MAIN CONTENT                │  DETAIL   │
│  (固定)  │              (可滚动)                    │  (抽屉)   │
│          │                                          │           │
└──────────┴──────────────────────────────────────────┴───────────┘
```

### 2.2 尺寸规范
- **左侧导航栏**: 240px 固定宽度，可折叠（折叠后 64px），支持垂直滚动
- **顶部 Header**: 64px 高度，深色/浅色双主题支持
- **主内容区**: 自适应宽度，padding 24px，卡片间距 16px
- **右侧详情抽屉**: 560px 宽度，支持半屏和全屏切换

### 2.3 Sidebar 导航结构

```
📊 概览               → Dashboard
🖥️ 设备管理           → 设备列表/连接管理

📂 业务网络配置       ──┐
  ├─ 🔌 接口配置     → BusinessInterface
  ├─ 🏷️ VLAN配置    → BusinessVlan
  └─ 🛣️ 路由配置     → BusinessRoute

⚙️ 原生配置           → 点击后加载动态子菜单
  ├─ [加载中...]      → 调用 CRD 接口 /api/crd/models
  ├─ huawei-if       ← 动态生成
  ├─ huawei-vlan     ← 动态生成
  └─ ...

📋 操作日志           → 审计日志
⚙️ 系统设置           → 平台配置
```

### 2.4 导航交互细节
- 分组菜单点击展开/折叠，默认展开状态
- 当前激活菜单项有左侧蓝色边框指示
- 原生配置菜单项初次点击时调用后端接口获取子菜单
- 折叠模式下只显示图标，分组名称 tooltip 显示

## 三、仪表盘页面设计

### 3.1 页面布局

```
┌───────────────────────────────────────────────────────────┐
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │ 设备总数 │ │ 在线设备 │ │ 配置同步 │ │ 今日操作 │    │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘    │
│                                                           │
│  ┌───────────────────────────────────────────────────┐   │
│  │ 设备状态分布图（环形图）                           │   │
│  └───────────────────────────────────────────────────┘   │
│                                                           │

└───────────────────────────────────────────────────────────┘
```

### 3.2 设计要点
- **顶部统计卡片**: 4个核心指标，带趋势箭头（较昨日）
- **设备状态图表**: ECharts 环形图，展示在线/离线/异常分布
- **最近操作表格**: 展示最近20条配置操作记录，支持跳转到对应设备
- **刷新机制**: 数据每30秒自动刷新，支持手动刷新
- **异常告警**: 设备离线或配置失败时，对应卡片显示红色告警状态

## 五、配置管理页面设计（动态渲染架构）

### 5.1 前后端协作架构

```
后端                                      前端
┌────────────────────┐          ┌──────────────────────────┐
│ YANG Schema        │  JSON    │ 通用控件映射表            │
│   ↓ 解析           │────────→│  - boolean → ElSwitch     │
│ SchemaRenderer     │          │  - enum → ElSelect       │
│   ↓ 输出           │          │  - string → ElInput      │
│ 控件描述元数据      │          │  - number → ElInputNumber│
└────────────────────┘          └──────────────────────────┘
```

### 5.2 控件元数据格式

```json
{
  "module": "openconfig-interfaces",
  "title": "接口配置",
  "fields": [
    {
      "path": "/interfaces/interface/name",
      "type": "string",
      "label": "接口名称",
      "placeholder": "例如: GigabitEthernet0/0/1",
      "required": true,
      "pattern": "^[a-zA-Z0-9/]+$",
      "readonly": false
    }
  ]
}
```

### 5.3 通用控件映射表

| YANG 类型 | 前端控件 | 说明 |
|-----------|----------|------|
| boolean | ElSwitch | 开关组件 |
| enumeration | ElSelect | 下拉选择 |
| string | ElInput | 文本输入 |
| int/uint | ElInputNumber | 数字输入 |
| leafref | ElSelect/Cascader | 关联引用 |
| list | ElTable | 动态表格 |
| container | ElCollapse | 分组面板 |

### 5.4 页面布局

```
┌───────────────────────────────────────────────────────────┐
│  选择设备: [设备下拉选择▼ 192.168.1.1        ] [刷新] │
├───────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────┐   │
│  │  (基于 YANG list 动态渲染的配置表格)              │   │
│  └───────────────────────────────────────────────────┘   │
│  [+ 新增配置项]                                           │
└───────────────────────────────────────────────────────────┘
```

### 5.5 提交流程
1. 点击提交
2. 前端校验（基于 YANG Schema 规则）
3. 显示配置差异预览
4. 用户确认
5. 下发到设备
6. 成功/失败 toast 提示

## 六、原生配置页面设计（CRD 动态驱动）

### 6.1 动态菜单加载流程

1. 初次点击 "原生配置" 菜单
2. 调用后端接口 `GET /api/crd/models?type=native`
3. 动态展开子菜单，显示所有模型名称
4. 缓存菜单数据，避免重复请求

### 6.2 后端接口设计

**GET /api/crd/models?type=native**
```json
{
  "models": [
    {
      "name": "huawei-if",
      "title": "华为接口配置",
      "vendor": "huawei",
      "description": "华为设备原生接口配置模型"
    }
  ]
}
```

### 6.3 组件复用
- 原生配置页面与业务配置页面 100% 复用同一套通用配置组件
- 仅通过 `module` 参数区分不同的 YANG 模块

## 七、操作日志和系统设置

### 7.1 操作日志页面
- 支持按时间范围、设备、操作类型、状态筛选
- 点击行查看操作详情（修改前后 diff、错误信息）
- 支持导出 CSV

### 7.2 系统设置页面
- 全局设置：缓存 TTL、同步间隔、超时时间、重试次数
- 主题设置：浅色模式 / 深色模式 / 跟随系统

## 八、技术实现要点

### 8.1 技术栈
- **框架**: Vue 3 + TypeScript + Vite
- **UI 组件库**: Element Plus
- **代码编辑器**: Monaco Editor（原生配置高级模式）
- **图表**: ECharts
- **HTTP 客户端**: Axios
- **状态管理**: Pinia（仅管理全局 UI 状态）

### 8.2 组件架构

```
src/
├── components/
│   ├── layout/
│   │   ├── MainLayout.vue      # 主布局容器
│   │   ├── Sidebar.vue         # 侧边导航（静态+动态混合）
│   │   ├── Header.vue          # 顶部栏
│   │   └── DetailDrawer.vue    # 右侧详情抽屉
│   ├── config/
│   │   ├── ConfigPage.vue      # 通用配置页面容器
│   │   ├── DynamicForm.vue     # 动态表单渲染器
│   │   ├── DynamicTable.vue    # 动态表格渲染器
│   │   └── FieldRenderer.vue   # 字段→控件映射器
│   └── dashboard/
├── stores/
│   ├── menu.ts                 # 动态菜单状态
│   └── device.ts               # 当前设备状态
├── api/
│   ├── crd.ts                  # CRD 接口
│   ├── device.ts               # 设备接口
│   └── config.ts               # 配置接口
└── utils/
    └── yang-schema.ts          # YANG Schema 解析工具
```

### 8.3 核心实现原则
1. **无状态设计**: 所有业务数据实时从后端拉取，前端仅存 UI 状态
2. **组件复用最大化**: 业务配置和原生配置 100% 复用同一套组件
3. **动态渲染优先**: 尽量避免手写固定表单
4. **类型安全**: TypeScript 严格模式
