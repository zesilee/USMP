# CLAUDE.md 分层结构化重写设计

> 日期: 2026-06-22
> 状态: Approved

## 目标

基于 Superpowers 技能体系，全自动优化 USMP 项目 CLAUDE.md：
1. **深度集成 worktree 安全合入**：Superpowers using-git-worktrees + finishing-a-development-branch 全流程
2. **OpenSpec 设计补齐**：五阶段表格化，每阶段有命令/产出/门禁
3. **存量项目渐进改造规范**：内置到 OpenSpec explore→propose→apply，禁止一次性重写
4. **AI 解析效率**：规则编号 + 表格 + 交叉引用，消除散文歧义

## 方案

**方案 A：分层结构化重写** — 5 层清晰结构，规则编号化，表格替代散文。

## 新 CLAUDE.md 结构

```
§1 项目身份          — 表格化定位 + 技术栈
§2 架构红线          — 12 条编号红线 R01-R12
§3 技术栈            — 5 层表格，约束列引用红线编号
§4 yang-controller-runtime 分层 — 组件编号 C1-C5
§5 开发工作流         — OpenSpec 五阶段 5.1-5.5 + TDD T01-T04 + 提交规范
§6 Worktree 安全隔离  — 创建→门禁→完成分支→清理→红线 W01-W06
§7 技能映射          — 后端/前端/Superpowers 三表
§8 数据存储          — 表格化
§9 异常处理          — 场景→策略→前端表现 三列表
§10 交付标准         — 维度→标准 二列表
```

## 关键设计决策

### 1. 规则编号系统

| 前缀 | 域 | 示例 |
|------|------|------|
| R01-R12 | 架构红线 | R03: 禁止数据库 |
| C1-C5 | yang-controller-runtime 组件 | C3: Reconciler |
| T01-T04 | TDD 规则 | T01: 测试先行 |
| W01-W06 | Worktree 红线 | W03: 禁止从内部 remove |
| 5.1-5.5 | 开发工作流阶段 | 5.3: apply |

AI 可直接引用编号替代重复描述全文，减少 token 消耗和歧义。

### 2. Worktree 深度集成

从 4 行散文扩展为完整 6 步流程：
- 6.1 创建：EnterWorktree → gitignore 验证 → 基线测试 → 记录对应关系
- 6.2 门禁：测试通过 + 代码评审 + 提交规范
- 6.3 完成分支：验证测试 → 检测环境 → 4 选 1 合入菜单
- 6.4 清理规则：路径溯源（.claude/worktrees/ → 自己清理，其他 → 禁止删除）
- 6.5 安全红线：6 条编号红线 W01-W06

### 3. 渐进改造内置到 OpenSpec

| 阶段 | 渐进改造要求 |
|------|-------------|
| explore | 审计存量代码，标记 legacy/新架构边界，输出改造策略 |
| propose | tasks.md 标注 legacy→新架构 迁移步骤，禁止一次性重写 |
| apply | 每步迁移：旧代码保留 + 新代码并行 + 双路径验证 → 切换 → 删除旧代码 |

### 4. 技能映射三层分类

- 7.1 后端技能：8 项，交叉引用 §4/R04/T02
- 7.2 前端技能：3 行（功能型/视觉型/不触发），显式列出"不触发"场景
- 7.3 Superpowers 技能：9 项，覆盖全开发生命周期

### 5. 结构化 vs 原版对比

| 维度 | 原版 | 新版 |
|------|------|------|
| 红线约束 | 散落在多个段落 | 12 条编号集中管理 |
| Worktree | 4 行散文 | 6 步流程 + 4 选 1 菜单 + 6 条红线 |
| 渐进改造 | 无 | 内置 3 阶段 |
| 技能映射 | 单表 11 行 | 三表 20 行 + Superpowers |
| 引用方式 | 重复描述 | 编号交叉引用 |
| 异常处理 | 5 行散文 | 5 行三列表 |
