# 提交前强制代码检视流程

## 步骤1：检查Git仓库状态
- 执行 `git status --porcelain` 查看是否有未提交的Go代码变更
- 如果无变更，输出：**当前无待提交代码，流程结束**

## 步骤2：如有变更，执行代码检视
使用 `go-code-review-check` 技能对所有修改的 `.go` 文件进行全面检视

### 检查项
1. **并发安全** - goroutine 泄漏、锁使用正确性
2. **ygot 类型安全** - 类型转换、空指针检查
3. **内存缓存安全** - TTL、LRU、并发访问
4. **NETCONF 异常处理** - 重连、超时、错误处理
5. **架构合规性** - 符合 yang-controller-runtime 规范
6. **代码规范** - 命名、注释、错误处理

## 步骤3：生成检视报告
必须输出完整的检视报告，包含：
- 检查文件列表
- 发现的问题列表（严重程度：高/中/低）
- 具体修改建议
- 最终结论（**通过** / **不通过**）

## 步骤4：根据检视结果处理

### 检视不通过（存在严重问题）
- 明确标记 **【检视不通过，禁止提交】**
- 列出必须修复的问题清单
- 自动对代码进行修复
- 修复完成后重新执行检视流程
- 循环直到检视通过

### 检视通过（无严重问题或仅低危建议）
- 标记 **【检视通过，允许提交】**
- 使用 `git-what-why-how-commit` 技能规范生成提交信息并自动提交
- 提交信息遵循三段式：**What**、**Why**、**Impact**（中文）

---

**重要提示**：代码检视是强制流程，必须先修复所有严重问题才能提交。每次迭代必须输出完整的检视报告供用户查阅。
