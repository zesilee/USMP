# 技能详情
## 一、激活时机（何时自动触发）
1.  当用户需求包含「代码评审」「Review」「代码检查」等关键词时，自动激活。
2.  开发流程中，任何业务代码编写完成后，自动触发本技能，执行全维度代码评审。
3.  代码评审不通过时，禁止触发 Commit 技能，需整改后重新评审。

## 二、核心原则（底层设计逻辑）
1.  合规性原则：严格对照项目全局规则和各技能规范，检查代码是否符合约束。
2.  安全性原则：重点检查并发安全、内存泄漏、协程泄露、panic 防护，避免系统隐患。
3.  规范性原则：检查代码格式、命名、注释，确保代码可读性、可维护性。
4.  实用性原则：评审结果需给出明确结论、风险点和整改建议，不做无意义检查。

风险点（重大）

无 panic 防护，NETCONF 连接失败时，协程崩溃可能影响整个 DeviceActor
整改建议
在 Connect () 方法中添加 sync.Mutex，确保并发安全
在重连协程、RPC 调用方法中添加 defer recover ()，捕获 panic 并打印日志
增加连接状态判断，避免重复连接
